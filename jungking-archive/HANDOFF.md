# JUNGKING 홈페이지 전시 자료 정리 인수인계

## 1. 사용자가 요청한 작업

사용자는 본인 홈페이지 `https://www.jungking.com`에 게시된 전시와 내용을 조사하고, 전시별 이미지를 폴더 단위로 정리해 달라고 요청했다.

홈페이지는 Wix로 제작되어 있으며 사용자가 Wix 계정에 직접 로그인한 뒤 편집기까지 열어 접근 권한을 제공했다. 비밀번호나 인증 정보는 전달받지 않았고, 로그인된 Chrome 탭을 통해 Wix 편집기와 미디어 관리자를 확인했다.

## 2. 수행한 작업

1. 공개 홈페이지의 `EXHIBITION` 메뉴에서 전시 항목 25개와 상세 URL을 확인했다.
2. 각 전시 페이지의 제목, 설명문, 실제 표시 이미지를 조사했다.
3. Wix 미디어 관리자에서 전체 미디어 구조를 확인했다.
4. 홈페이지 전시 페이지에 실제 사용된 이미지와 Wix 원본 파일 ID를 대조했다.
5. 전시별 로컬 폴더를 생성하고 Wix 원본 이미지를 다운로드했다.
6. 각 전시 폴더에 홈페이지 주소와 설명을 `README.md`로 기록했다.
7. 다운로드 실패, 빈 파일, 이미지 형식 손상 여부를 검수했다.

Wix 사이트의 페이지 내용이나 파일 배치는 수정하지 않았다. 읽기와 다운로드만 수행했다.

## 3. 데이터 저장 위치

전체 작업물:

`/Users/jk/Documents/힉스필드1/jungking_archive`

주요 파일:

- `README.md`: 전체 아카이브 안내
- `HANDOFF.md`: 이 인수인계 문서
- `manifest.json`: 전시별 제목, URL, 설명, 이미지 파일명, Wix 미디어 ID, 원본 URL
- `download_archive.mjs`: `manifest.json`을 기준으로 전시 폴더와 설명 파일을 만들고 원본 이미지를 다운로드하는 스크립트
- `_source/wix_media_inventory.json`: Wix 미디어 관리자에서 수집한 전체 미디어 파일 인벤토리
- `_source/download_failures.json`: 다운로드 실패 기록. 현재 내용은 빈 배열 `[]`

## 4. 폴더 구성

전시 폴더는 홈페이지 메뉴 순서대로 `01_`부터 `25_`까지 번호를 붙였다.

예:

```text
jungking_archive/
├── 01_2025 2인전, 갤러리오로라/
│   ├── README.md
│   ├── 001_DSC00056.JPG
│   ├── 002_DSC00078.JPG
│   ├── 003_DSC00058.JPG
│   └── 004_C0004T01.JPG
├── 02_2024 인천아트쇼,갤러리 위/
├── ...
├── 25_2004 RESFEST 영화제 서울/
├── _source/
├── manifest.json
├── download_archive.mjs
├── README.md
└── HANDOFF.md
```

각 전시 폴더의 `README.md`에는 다음이 들어 있다.

- 전시 제목
- 원본 홈페이지 URL
- 분류 상태
- 이미지 수
- 홈페이지에 표시된 제목과 설명문
- 문제가 있는 경우 확인 경고

이미지 파일 앞에는 홈페이지 표시 순서에 따라 `001_`, `002_` 같은 번호를 붙였다. 뒤쪽 이름은 가능한 경우 Wix에 등록된 원래 파일명을 유지했다.

## 6. 반드시 확인할 문제

`06_2023 Collabo Collage Gallery,2인전,울산`

홈페이지 URL:

`https://www.jungking.com/2023collabocollagegallery`

이 주소는 현재 2023 울산 전시가 아니라 2025 갤러리오로라 메인 콘텐츠를 표시한다. 잘못된 사진을 2023 자료로 확정하지 않기 위해 이 폴더에는 이미지 없이 설명과 경고만 남겼다.

다음 작업자는 사용자가 2023 울산 전시의 올바른 Wix 페이지나 원본 사진을 지정하기 전까지 2025 이미지를 이 폴더에 넣으면 안 된다.

## 7. 데이터 사용 방법

사람이 자료를 확인할 때는 전시별 폴더와 해당 폴더의 `README.md`를 보면 된다.

프로그램이나 다른 사이트로 이전할 때는 `manifest.json`을 기준 데이터로 사용한다. 각 전시 객체에는 다음 필드가 있다.

```text
order       메뉴 순서
title       전시명
folder      로컬 폴더명
url         홈페이지 상세 페이지
headings    페이지 제목
paragraphs  전시 설명문
assets      이미지 파일명, Wix ID, 원본 URL
status      ready 또는 review_required
```

Wix 전체 파일에서 추가 자료를 찾을 때는 `_source/wix_media_inventory.json`의 `name`, `mediaId`, `originalUrl`을 검색한다.

## 8. 재실행 및 주의사항

원본 다운로드를 다시 실행하려면 다음 명령을 사용한다.

```bash
node "/Users/jk/Documents/힉스필드1/jungking_archive/download_archive.mjs"
```

이 스크립트는 네트워크 접근이 필요하며 `manifest.json`에 들어 있는 원본 URL을 사용한다. 기존 파일을 동일한 이름으로 다시 내려받을 수 있으므로, 수동 수정한 이미지가 있다면 실행 전에 백업해야 한다.

`manifest.json`의 2023 항목에는 홈페이지에서 잘못 노출된 이미지 정보가 남아 있을 수 있지만 상태가 `review_required`이므로 다운로드 스크립트는 해당 전시 이미지를 생성하지 않는다. 후속 자동화에서도 반드시 `status`를 확인해야 한다.

## 9. 후속 작업 우선순위

1. 2023 울산 전시의 정확한 Wix 페이지 또는 원본 이미지 확인
2. 확인된 2023 이미지를 폴더와 `manifest.json`에 추가
3. 필요할 경우 전시 데이터를 새 홈페이지, 포트폴리오 DB 또는 스프레드시트 형식으로 변환
4. Wix 편집이 필요하면 변경 전 백업하고 사용자 확인 후 게시
